### Customer Journey Map (CJM) — базовый сценарий
| Этап | Действие пользователя | Эмоция | Боль | Возможность |
| --- | --- | --- |--- |--- |
| 1 | Открывает приложение | нейтрально | не помнит, что делал | быстрый вход |
| 2 | Видит список задач | спокойствие | много невыполненных | фильтр по важности |
| 3 | Добавляет новую задачу | удовлетворение | долгая форма | одна строка + Enter |
| 4 | Отмечает задачу | радость | нет прогресса | визуальный счётчик |
| 5 | Закрывает приложение | легкость | страх забыть | автосохранение

### Первые бизнес-требования (СА)
**|BR-01:** Пользователь может зарегистрироваться по email и паролю.
**BR-02:** Пользователь может войти в систему.
**BR-03:** Пользователь может создать задачу с заголовком (обязательно) и опционально — датой выполнения.
**BR-04:** Пользователь может отметить задачу как выполненную.
**BR-05:** Пользователь может редактировать задачу.
**BR-06:** Пользователь может удалить задачу.
**BR-07:** Пользователь может фильтровать задачи: все / активные / выполненные.
**BR-08:** Задачи привязаны к конкретному пользователю (нет общих списков).

### Техническая реализуемость (BE)
Стек:
•	Backend: Python + FastAPI (лёгкий, быстрый)
•	База данных: SQLite (на старте) / PostgreSQL (при деплое)
•	Frontend: React
•	Аутентификация: JWT
•	Деплой: Railway / Render (бесплатно)
